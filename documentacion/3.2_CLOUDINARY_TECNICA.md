## 1. Descripción General

Cloudinary es el servicio externo utilizado para **almacenar y servir las imágenes personalizadas** del sistema (header y footer de la página de registro). Reemplaza a Firebase Storage en este rol porque:

| Aspecto | Firebase Storage (descartado) | Cloudinary (adoptado) |
|---------|------------------------------|-----------------------|
| Plan gratuito | Requiere Blaze (pago) | Free tier generoso (25 GB) |
| URLs públicas | Necesitan firma o reglas específicas | `secure_url` siempre pública |
| Upload sin backend | Complejo con Security Rules | Upload Preset sin autenticación |
| CDN global | No incluido en free | Integrado |
| Transformaciones | No disponibles | Resize, format, quality on-the-fly |

## 2. Cuenta y Configuración

### Credenciales de acceso

| Parámetro | Valor | Ubicación en código |
|-----------|-------|---------------------|
| Cloud Name | `casinomagic` | `CLOUDINARY_CLOUD_NAME` |
| Upload Preset | `registro_qr` | `CLOUDINARY_UPLOAD_PRESET` |
| Carpeta destino | `configuracion` | `CLOUDINARY_FOLDER` |

> **Acceso al dashboard:** [https://cloudinary.com/console](https://cloudinary.com/console)  
> Credenciales de login gestionadas por el área de IT de Casino Magic.

### Definición en `admin.html`

```javascript
// Configuración de Cloudinary
const CLOUDINARY_CLOUD_NAME = 'casinomagic';
const CLOUDINARY_UPLOAD_PRESET = 'registro_qr';
const CLOUDINARY_FOLDER = 'configuracion';
```

Estas constantes se declaran al inicio del bloque `<script>` principal de `admin.html`, dentro del scope global de la página.

### ¿Qué es un Upload Preset?

Un **Upload Preset** es una configuración guardada en Cloudinary que define las reglas de subida (carpeta destino, transformaciones aplicadas, modo de acceso, etc.) sin exponer la API Secret en el cliente.

El preset `registro_qr` está configurado como **unsigned** (no requiere firma), permitiendo subidas directas desde el navegador sin pasar por un backend propio.

## 3. Flujo de Subida de Imágenes

```
Usuario selecciona archivo (input[type="file"])
         │
         ▼
validarImagen(file)
  ├── Verifica formato: JPG / PNG / WebP
  ├── Verifica tamaño: máximo 2 MB

uploadToCloudinary(file, type)

         │
         ▼
config-loader.js en index.html lee la URL de Firestore
y la aplica en <img id="headerImg"> y <img id="footerImg">
```

## 4. API Reference — Endpoint utilizado

### Upload API (firmado / no firmado)

```
POST https://api.cloudinary.com/v1_1/{cloud_name}/image/upload
```

**URL efectiva en este proyecto:**
```
POST https://api.cloudinary.com/v1_1/casinomagic/image/upload
```

### Parámetros enviados (`FormData`)

| Campo | Tipo | Valor | Descripción |
|-------|------|-------|-------------|
| `file` | File | Archivo del input | Imagen binaria a subir |
| `upload_preset` | string | `registro_qr` | Preset unsigned configurado en Cloudinary |
| `folder` | string | `configuracion` | Subcarpeta dentro del Media Library |
| `public_id` | string | `header-{timestamp}` o `footer-{timestamp}` | Identificador único del asset |

### Respuesta exitosa (HTTP 200)

```json
{
  "asset_id": "abc123...",
  "public_id": "configuracion/header-1709123456789",
  "version": 1709123456,
  "version_id": "xyz...",
  "signature": "...",
  "width": 1920,
  "height": 200,
  "format": "jpg",
  "resource_type": "image",
  "created_at": "2026-03-04T14:30:00Z",
  "tags": [],
  "bytes": 245678,
  "type": "upload",
  "url": "http://res.cloudinary.com/casinomagic/image/upload/v.../...",
  "secure_url": "https://res.cloudinary.com/casinomagic/image/upload/v.../configuracion/header-1709123456789.jpg"
}
```

> El sistema utiliza **únicamente** el campo `secure_url` (HTTPS) descartando el resto de la respuesta.

### Respuesta de error (HTTP 4xx)

```json
{
  "error": {
    "message": "Upload preset not found"
  }
}
```

## 5. Código de Implementación

### Función `uploadToCloudinary` — `admin.html`

```javascript
/**
 * Sube una imagen a Cloudinary usando el Upload Preset unsigned.
 * @param {File} file   - Objeto File del input (ya validado)
 * @param {string} type - 'header' o 'footer' (afecta el public_id)
 * @returns {Promise<string>} URL pública HTTPS de la imagen subida
 */
async function uploadToCloudinary(file, type) {
    const formData = new FormData();
    formData.append('file', file);
    formData.append('upload_preset', CLOUDINARY_UPLOAD_PRESET);
    formData.append('folder', CLOUDINARY_FOLDER);
    formData.append('public_id', `${type}-${Date.now()}`);
    //  Ejemplo: "header-1709123456789" o "footer-1709123456789"

    const response = await fetch(
        `https://api.cloudinary.com/v1_1/${CLOUDINARY_CLOUD_NAME}/image/upload`,
        {
            method: 'POST',
            body: formData
        }
    );

    if (!response.ok) {
        const error = await response.json();
        throw new Error(error.error?.message || 'Error subiendo imagen a Cloudinary');
    }

    const data = await response.json();
    return data.secure_url; // Única propiedad que se usa
}
```

### Llamada desde `guardarPersonalizacion`

```javascript
async function guardarPersonalizacion(event) {
    event.preventDefault();
    const overlay = document.getElementById('loadingOverlay');
    overlay.style.display = 'flex'; // Mostrar spinner

    try {
        const config = {
            primaryColor: document.getElementById('inputPrimaryColor').value,
            headerBackgroundColor: useDefaultGradient ? '' : document.getElementById('inputHeaderBgColor').value,
            cacheVersion: Date.now(),
            updatedAt: firebase.firestore.FieldValue.serverTimestamp(),
            updatedBy: firebase.auth().currentUser.email
        };

        // Header — solo sube si el usuario seleccionó un archivo nuevo
        const headerFile = validarImagen(document.getElementById('inputHeaderImage').files[0]);
        if (headerFile) {
            config.headerImage = await uploadToCloudinary(headerFile, 'header');
        }

        // Footer — solo sube si el usuario seleccionó un archivo nuevo
        const footerFile = validarImagen(document.getElementById('inputFooterImage').files[0]);
        if (footerFile) {
            config.footerImage = await uploadToCloudinary(footerFile, 'footer');
        }

        // Guardar en Firestore (merge para no pisar campos no incluidos)
        await db.collection('configuracion').doc('estilos').set(config, { merge: true });

    } catch (error) {
        alert('❌ Error guardando la configuración: ' + error.message);
    } finally {
        overlay.style.display = 'none';
    }
}
```

## 6. Validación de Archivos

Los archivos son validados **antes** de enviarse a Cloudinary mediante la función `validarImagen`:

```javascript
/**
 * Valida un archivo de imagen antes de subir a Cloudinary.
 * @param {File|undefined} file - Archivo del input
 * @returns {File|null} - El mismo file si es válido, null si no pasa validación
 */
function validarImagen(file) {
    const MAX_SIZE = 2 * 1024 * 1024; // 2 MB en bytes
    const VALID_TYPES = ['image/jpeg', 'image/jpg', 'image/png', 'image/webp'];

    if (!file) return null; // No hay archivo seleccionado

    if (!VALID_TYPES.includes(file.type)) {
        alert('❌ Formato no válido. Usa JPG, PNG o WEBP');
        return null;
    }

    if (file.size > MAX_SIZE) {
        const sizeMB = (file.size / 1024 / 1024).toFixed(2);
        alert(`❌ Archivo muy grande (${sizeMB}MB). Máximo 2MB`);
        return null;
    }

    return file; // Archivo válido
}
```

### Reglas de validación

| Regla | Detalle |
|-------|---------|
| Formatos aceptados | `image/jpeg`, `image/jpg`, `image/png`, `image/webp` |
| Tamaño máximo | 2 MB (2.097.152 bytes) |
| Validación de tipo | Basada en `file.type` (MIME type del navegador) |
| Validación doble | También hay validación secundaria en el evento `change` del input en `previewImage()` |

> **Nota:** La validación es del lado del cliente (frontend). Cloudinary también aplica sus propias restricciones server-side según el Upload Preset.

## 7. Nomenclatura de Archivos (public_id)

El `public_id` identifica unívocamente cada asset dentro de Cloudinary:

```javascript
formData.append('public_id', `${type}-${Date.now()}`);
```

### Ejemplos generados

| Tipo | Timestamp (ms) | public_id resultante | Ruta completa en Cloudinary |
|------|----------------|----------------------|-----------------------------|
| header | 1709500000000 | `header-1709500000000` | `configuracion/header-1709500000000` |
| footer | 1709500000123 | `footer-1709500000123` | `configuracion/footer-1709500000123` |

### ¿Por qué timestamp?

- Garantiza **unicidad** sin necesidad de un UUID externo.
- Evita colisiones aunque se suban dos headers en la misma sesión.
- Permite ordenar cronológicamente desde el Media Library de Cloudinary.
- Las imágenes anteriores **no se sobreescriben** y permanecen en Cloudinary (no se eliminan automáticamente). Si se desea gestionar versiones, hacerlo manualmente desde el dashboard.

## 8. Persistencia en Firestore

Las URLs de Cloudinary se guardan en el documento `configuracion/estilos` de Firestore:

### Estructura del documento `estilos`

```json
{
  "primaryColor": "#a69867",
  "headerBackgroundColor": "",
  "headerImage": "https://res.cloudinary.com/casinomagic/image/upload/v1709123456/configuracion/header-1709123456789.jpg",
  "footerImage": "https://res.cloudinary.com/casinomagic/image/upload/v1709123456/configuracion/footer-1709123456789.jpg",
  "cacheVersion": 1709123456789,
  "updatedAt": "2026-03-04T14:30:00Z",
  "updatedBy": "admin@casinomagic.com.ar"
}
```

### Campos clave

| Campo | Tipo | Descripción |
|-------|------|-------------|
| `headerImage` | string (URL) | URL HTTPS de la imagen de header en Cloudinary. Vacío si usa imagen local por defecto. |
| `footerImage` | string (URL) | URL HTTPS de la imagen de footer en Cloudinary. Vacío si usa imagen local por defecto. |
| `cacheVersion` | number | Timestamp usado para invalidar caché en los clientes. |

### Comportamiento con `merge: true`

```javascript
await db.collection('configuracion').doc('estilos').set(config, { merge: true });
```

Con `merge: true`, solo se actualizan los campos incluidos en `config`. Si el usuario no seleccionó imagen de header, el campo `headerImage` no se incluye y **la URL anterior se preserva**.

## 9. Consumo en el Frontend Público

El archivo `js/config-loader.js` carga la configuración desde Firestore al iniciar `index.html` y aplica las imágenes de Cloudinary al header y footer de la página de registro:

```javascript
// Fragmento conceptual de config-loader.js
async function cargarConfiguracion() {
    const db = firebase.firestore();
    const doc = await db.collection('configuracion').doc('estilos').get();

    if (doc.exists) {
        const config = doc.data();

        // Header
        if (config.headerImage) {
            document.getElementById('headerImg').src = config.headerImage;
        }

        // Footer
        if (config.footerImage) {
            document.getElementById('footerImg').src = config.footerImage;
        }

        // Color primario
        if (config.primaryColor) {
            document.documentElement.style.setProperty('--primary-color', config.primaryColor);
        }
    }
}
```

### Fallback a imágenes locales

Si `headerImage` o `footerImage` están vacíos (no se configuró imagen personalizada), se mantienen las imágenes locales del proyecto:

| Tipo | Imagen por defecto |
|------|--------------------|
| Header | `img/Heather.png` |
| Footer | `img/Pie blanco.png` |

## 10. Seguridad

### Upload Preset unsigned

El preset `registro_qr` es **sin firma (unsigned)**, lo que significa que el `cloud_name` y el `upload_preset` son visibles en el código del cliente. Esto es **intencional y aceptable** bajo las siguientes consideraciones:

| Riesgo | Mitigación |
|--------|------------|
| Cualquiera puede subir imágenes al preset | El preset puede configurarse con restricciones de origen (Allowed Origins) en el dashboard |
| Abuso del almacenamiento | Límite de tamaño 2 MB aplicado server-side en el preset |
| Imágenes maliciosas | Cloudinary escanea contenido en uploads (moderación automática) |
| Exposición del cloud_name | No es un secreto, es equivalente a un nombre de usuario público |

> **Recomendación:** Configurar **Allowed Domains** en el preset `registro_qr` desde el dashboard de Cloudinary para que solo acepte uploads desde el dominio de Vercel (`registroqr.vercel.app`).

### Lo que NO se expone

- ✅ API Key de Cloudinary → no está en el código del cliente
- ✅ API Secret de Cloudinary → no está en el código del cliente
- ✅ Credenciales de Firebase Admin → no están en el cliente
- ✅ Credenciales de login de administrador → manejadas por Firebase Auth

## 11. Plan Gratuito — Límites y Cuotas

Cloudinary Free Tier (a Marzo 2026):

| Recurso | Límite Free |
|---------|-------------|
| Almacenamiento | 25 GB |
| Ancho de banda | 25 GB / mes |
| Transformaciones | 25 créditos/mes |
| Uploads | Ilimitados (dentro del almacenamiento) |
| Soporte | Comunidad |

### Estimación de uso para este proyecto

Cada imagen pesa aproximadamente 100–500 KB. Con el plan gratuito se pueden subir aproximadamente **50.000 imágenes** sin riesgo de superar los 25 GB. Para este caso de uso (pocas imágenes de header y footer por evento), el plan gratuito es ampliamente suficiente.

## 12. Gestión de Imágenes desde el Dashboard

### Acceso

1. Ir a [https://cloudinary.com/console](https://cloudinary.com/console)
2. Login con credenciales del área de IT
3. Ir a **Media Library** → carpeta `configuracion`

### Operaciones disponibles desde el dashboard

| Operación | Descripción |
|-----------|-------------|
| Ver imágenes activas | Ver todas las imágenes subidas en `configuracion/` |
| Eliminar imágenes antiguas | Seleccionar y borrar versiones previas de header/footer |
| Renombrar | Cambiar public_id (requiere actualizar URL en Firestore también) |
| Ver detalles | Tamaño, formato, dimensiones, fecha de subida |
| Copiar URL | Obtener `secure_url` para pegar manualmente en Firestore |

> **Importante:** Si se elimina una imagen desde el dashboard y esa URL está guardada en Firestore, el header o footer aparecerá roto en la página de registro hasta que se suba una nueva imagen.

## 13. Errores Comunes y Soluciones

| Error | Causa probable | Solución |
|-------|---------------|----------|
| `Upload preset not found` | El preset `registro_qr` no existe en la cuenta | Crear el preset desde Cloudinary Console → Settings → Upload |
| `Invalid cloud_name` | El cloud name `casinomagic` está mal escrito | Verificar en Cloudinary Console → Account Details |
| `File size too large` | El preset tiene límite configurado | Ajustar `max_file_size` en el preset, o reducir la imagen antes de subir |
| `Invalid image format` | El archivo no es una imagen real (aunque tenga extensión .jpg) | El usuario debe subir un archivo de imagen válido |
| `Network error` | Sin conexión a internet durante el upload | Verificar conexión y reintentar |
| Imagen no aparece en la página | Se eliminó la imagen de Cloudinary | Subir nueva imagen desde el admin panel |
| Header/footer desactualizado | Cache del navegador | Forzar recarga con `Ctrl+F5` (el campo `cacheVersion` ayuda a invalidar el caché programáticamente) |
